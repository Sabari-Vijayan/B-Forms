# Project Structure

## 📂 Packages Overview

### `@workspace/api-server` (`artifacts/api-server`)
The main backend application.
* `src/routes/`: Express route handlers grouped by domain (forms, fields, submissions, etc.).
* `src/lib/`: Shared utilities (AI, Auth, Logger, Supabase).
* `build.mjs`: esbuild configuration for production bundling.

### `@workspace/prompt-to-form` (`artifacts/prompt-to-form`)
The main React frontend application.
* `src/pages/`: Main application views (Dashboard, CreateForm, PublicForm).
* `src/components/`: Reusable UI components, including Shadcn UI primitives.
* `src/hooks/`: Custom React hooks, including those generated by Orval.

### `@workspace/mockup-sandbox` (`artifacts/mockup-sandbox`)
A separate Vite environment for isolated UI component development. Useful for prototyping complex forms or UI interactions without running the full app.

### `lib/api-spec`
Contains the OpenAPI 3.1 specification (`openapi.yaml`). This is the **source of truth** for all API communication.

### `lib/api-zod`
Contains generated Zod schemas based on the OpenAPI spec. These are used by both the backend (for request validation) and the frontend.

### `lib/api-client-react`
Contains generated React hooks (TanStack Query) based on the OpenAPI spec. These are used by the frontend to interact with the API in a type-safe way.

### `lib/db`
Contains the Drizzle ORM configuration and database connection logic. While migrations are currently managed via SQL files, this package provides the TypeScript types for database rows.
